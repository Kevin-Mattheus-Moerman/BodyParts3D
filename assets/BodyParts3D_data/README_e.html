BodyParts3D

2013/04/11

WebSite: http://lifesciencedb.jp/bp3d/?lng=en
FTP Site: ftp://ftp.biosciencedbc.jp/archive/bodyparts3d/

BodyParts3D is a dictionary-type database for anatomy in which shapes and positions of body parts are represented by 3D human models. This directory contains the models labeled with anatomical terms in BodyParts3D.

2.3 Table of 3D organ model IDs and organ names

Data name Table of 3D organ model IDs and organ names
Description of data contents

List of downloadable 3D organ models in a tab-delimited text file format, describing the correspondence between 3D organ model IDs and organ names

File parts_list_e.txt (49KB)

Each column is delimited by a tab character in the file.
Data items are the following:
Data item Description
id ID of the corresponding FMA (Foundational Model of Anatomy), starting with "FMA". If a corresponding concept is not found in the FMA, a BodyParts3D's original ID that begins with "BP" is assigned.
en English name. Basically, a preferred name of FMA is assigned.

2.4 Data describing the inclusion (part-of) relations between two organs

Data name Data describing the inclusion relations between two organs
Description of data contents

File of data describing the inclusion relations between two organs of the downloadable 3D organ models. This is the conventional type hierarchy shown at the "Tree" tab of "BP3D Viewer" (http://lifesciencedb.jp/bp3d).

File conventional_part_of.txt (137KB)

Each column is delimited by a tab character in the file.
Data items are the following:
Data item Description
id Organ ID
name English name of the organ in the first column
part id ID of an organ that is part of the organ in the first and second columns
part name English name of the organ in the third column (part id)

2.5 Data describing the definition of composite organs

Data name Data describing the definition of composite organs
Description of data contents

In BodyParts3D, only atomic organs are constructed as polygon meshes. Composite organs defined as a sum of atomic ones are not provided in the form of the meshes. This file describes which atomic body parts constitute each composite organ according to the "conventional" inclusion relations described above.

File composite_parts.txt (721KB)

Each column is delimited by a tab character in the file.
Data items are the following:
Data item Description
composite id Composite organID
composite name English name of the organ in the first column
primitive id ID of an atomic organ that constitutes the organ in the first and second columns
primitive name English name of the organ in the third column (primitive id)

2.6 Organ ID-specific polygon mesh data

2.6.1 Polygon reduction rate = 95.0%
Data name Organ ID-specific polygon mesh data 1 (Polygon reduction rate = 95.0%)
Description of data contents

BodyParts3D phase-2 or 3 3D organ model data with the polygon reduction rate of 95%. The zip-compressed download files contain multiple files of organ ID-specific polygon data in the Wavefront OBJ format. The polygon data file is named as "organ ID" of the 3D organ models followed by the extension ".obj".

File BodyParts3D_3.0_obj_95.zip (521MB)

2.7 Organ ID-specific polygon mesh data 2 (Polygon reduction rate = 99.0%)

Data name Organ ID-specific polygon mesh data 2 (Polygon reduction rate = 99.0%)
Description of data contents

BodyParts3D phase-2 or 3 3D organ model data with the polygon reduction rate of 99%. The zip-compressed download files contain multiple files of organ ID-specific polygon data in the Wavefront OBJ format. The polygon data file is named as "organ ID" of the 3D organ models followed by the extension ".obj".

File BodyParts3D_3.0_obj_99.zip (127MB)
About polygon data

A polygon mesh with the reduction rate of 95% is more precise than that with 99% because 95% has five times more polygons (triangles) than 99%.
